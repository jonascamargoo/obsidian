A prototipagem é uma técnica utilizada no desenvolvimento de software para criar modelos ou representações preliminares de um sistema ou parte dele, com o objetivo de validar requisitos, explorar funcionalidades, testar conceitos e obter feedback dos usuários ou clientes. Esses modelos, conhecidos como protótipos, podem variar em complexidade, desde simples esboços até versões mais detalhadas e interativas do sistema.

### Quando é utilizada?

1. Há dificuldades no entendimento dos requisitos do sistema:
    - Quando os requisitos não estão claros ou são ambíguos, a construção de protótipos pode ajudar a esclarecer as necessidades do usuário e a visualizar como o sistema deverá funcionar na prática.
    
2. Há requisitos que precisam ser mais bem entendidos:
    - Quando certos requisitos são complexos ou difíceis de especificar completamente no início do projeto, a prototipagem pode ser útil para explorar esses requisitos em mais detalhes e compreendê-los melhor


### Quais as fases?

1. Após o Levantamento de Requisitos (LR), um protótipo é construído para ser usado na validação:
    - Com base nos requisitos levantados, é criado um protótipo que demonstre as principais funcionalidades ou aspectos do sistema.
    
2. Usuários fazem críticas:
    - O protótipo é apresentado aos usuários, que fornecem feedback e críticas sobre sua usabilidade, funcionalidade e adequação aos requisitos.
    
3. O protótipo é então corrigido ou refinado:
    - Com base no feedback dos usuários, o protótipo é ajustado e refinado para abordar as críticas e melhorar sua qualidade.
    
4. O processo de revisão e refinamento continua até que o protótipo seja aceito:
    - O ciclo de revisão e refinamento é repetido várias vezes até que o protótipo atenda às expectativas dos usuários e aos requisitos do sistema.
    
5. Após a aceitação, o protótipo é descartado ou utilizado como uma versão inicial do sistema:
    - Uma vez que o protótipo seja aprovado, ele pode ser descartado e o desenvolvimento do sistema pode prosseguir com base nas lições aprendidas durante a prototipagem, ou o protótipo pode ser evoluído para se tornar a versão inicial do sistema, servindo como uma base sólida para o desenvolvimento adicional.


###### Considerações

- Note que a prototipagem NÃO é um substituto à construção de modelos do sistema;
- A prototipagem é uma técnica complementar à construção dos modelos do sistema;
- Mesmo com o uso de protótipos, os modelos do sistema devem ser construídos;
- Os erros detectados na validação do protótipo devem ser utilizados para modificar e refinar os modelos do sistema.
