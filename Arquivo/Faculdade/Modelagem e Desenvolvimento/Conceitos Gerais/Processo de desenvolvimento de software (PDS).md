## O que é?
  
O Processo de Desenvolvimento de Software (PDS) é uma abordagem sistemática para planejar, criar, testar e manter sistemas de software. Ele fornece uma estrutura para guiar as atividades envolvidas no desenvolvimento de software, desde a concepção até a entrega e manutenção do produto final. O objetivo principal do PDS é garantir que o software seja desenvolvido de maneira eficiente, com alta qualidade e dentro do prazo e do orçamento previstos.
## Etapas

O PDS geralmente envolve uma série de etapas sequenciais e iterativas, que podem incluir:

1. **Planejamento**: Nesta fase, os requisitos do sistema são identificados e analisados, os objetivos do projeto são definidos e um plano geral é elaborado. Isso pode envolver a identificação de stakeholders, a definição de metas de negócios e a avaliação dos recursos necessários para o projeto.
    
2. **Análise de requisitos**: Durante esta fase, os requisitos funcionais e não funcionais do sistema são detalhados e documentados. Isso inclui a identificação de casos de uso, diagramas de classes, diagramas de sequência, entre outros artefatos.
    
3. **Projeto**: Nesta etapa, o sistema é projetado com base nos requisitos identificados na fase anterior. Isso pode incluir o design arquitetural, o design de banco de dados, o design de interface do usuário e outros aspectos técnicos do sistema.
    
4. **Implementação**: Durante esta fase, o código-fonte do sistema é desenvolvido de acordo com os designs criados na etapa de projeto. Os desenvolvedores escrevem, testam e depuram o código para garantir que ele atenda aos requisitos especificados.
    
5. **Testes**: Após a implementação, o sistema é testado para garantir que funcione conforme o esperado e que atenda aos requisitos estabelecidos. Isso pode incluir testes de unidade, testes de integração, testes de sistema e testes de aceitação do usuário.
    
6. **Implantação**: Nesta fase, o sistema é preparado e implantado no ambiente de produção. Isso pode envolver a instalação do software em servidores, a migração de dados e a configuração do ambiente de produção.
    
7. **Manutenção**: Após a implantação, o sistema entra em uma fase de manutenção, onde são feitas correções de bugs, melhorias de desempenho e atualizações de funcionalidades conforme necessário.
    

	OBS: modelagem de sistemas é focada nas etapas 2 e 3, análise de requisitos e projeto.

#### Exemplos de PDS

- ICONIX
- RUP (Rational Unified Process)
- EUP (Enterprise Unified Process)
- XP (Extreme Programming) 
- OPEN (Object oriented Process, Environment and Notation)
